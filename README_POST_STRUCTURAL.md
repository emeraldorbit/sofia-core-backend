## The Post-Structural Runtime

The runtime integrates all three movements into a cohesive, living system.

### Basic Usage

```typescript
import { createPostStructuralRuntime } from './post_structural';

// Create runtime
const runtime = createPostStructuralRuntime();

// Activate the post-structural sequence
runtime.activate();

// Execute a cycle (automatically uses current movement)
const result = runtime.cycle();

// Or use specific movements
const expr = runtime.express();
const recur = runtime.recurse();
const ident = runtime.identify('decision');

// Unify all three movements in one operation
const unified = runtime.unify();
// Returns: { expression, recursion, identity }

// Get state
const state = runtime.getState();
// state.movement: 'expression' | 'recursion' | 'identity' | 'unified'
// state.active: true/false
// state.coherence: 0.0 - 1.0
// state.sovereign: true/false
```

### Global Runtime

For application-wide access:

```typescript
import { getPostStructuralRuntime } from './post_structural';

// Get or create the global runtime (singleton)
const runtime = getPostStructuralRuntime();

// The runtime is automatically activated
runtime.express();
runtime.recurse();
runtime.identify('action');
```
